Last Light · Demo Build

Demo Plan & Vertical Slice

This page outlines the focused demo build for Last Light – a mix of zombie survival and farm/base build simulation where the goal is to build a home base that can both defend against zombies and sustain a functioning farm.

View Demo Scope Open Docs Library

Demo at a Glance

Core Loop
Farm and build defenses by day; defend the base from zombies at night.
Genre
Zombie survival · Farm & base build simulation
Camera
Same camera style as classic Harvest Moon.
Goal of Demo
5–10 minutes of structured play that proves the core loop and base-defense fantasy.

Scope

What the Demo Includes

The demo focuses on a tight slice of the full game loop: gathering basic resources, establishing a simple farm inside the base, and surviving escalating zombie pressure.

The objectives for the player in this vertical slice are:

  • Build and maintain a small, defensible home base.
  • Set up a functioning farm inside the base to supply resources.
  • Survive wandering zombies and basic night waves as difficulty increases.

Concrete Features in the Demo

  • Starter weapon: Axe crafted from an easy-to-gather stick that breaks very quickly.
  • Ranged options: Simple handgun and crossbow for basic defense.
  • Crops: A small set of crops with planting, watering, and harvesting loops.
  • Defenses: Wooden fence, scarecrow defense, basic traps, and a basic turret.
  • Enemies: Constant wandering zombies, with simple waves that increase in count at night.
  • World slice: One small forest area and one small town connected to the base.

Production

Demo Milestones

The demo is treated as a vertical slice with clear, gated milestones from design through investor-ready build and materials.

  1. 1. Demo Design & GDD Update

    Update the Last Light Game Design Document to clearly define the demo scope, core mechanics to validate, and win/fail conditions.

  2. 2. Core Loop Prototype

    Implement basic day/night cycle, farming, base building, and a simple zombie encounter to validate “build your base, defend at night” as the core loop.

  3. 3. Visual Direction Lock

    Establish pixel-art style, base/farm tiles, zombies, and key UI elements for the demo slice.

  4. 4. Vertical Slice Build (5–10 Minutes)

    Polish a contained map with the small forest, small town, home base, and one complete day/night cycle that showcases farming, building, and defending.

  5. 5. Pitch Materials

    Create an investor-facing package: updated GDD extract, business one-pager, and short gameplay capture to accompany the demo build.

  6. 6. Publisher & Investor Outreach

    Use the demo and materials in targeted outreach as the foundation for full-game funding conversations.

Budget

Cost Targets for the Demo

The demo is budgeted as a lean vertical slice, sitting on top of a very small studio setup cost.

Studio Setup Goal

$800 – $3,000

Basic studio formation and tools (core hardware, software, and minimal overhead).

Demo Production Goal

$4,000 – $10,600

Demo development, art, sound, QA, and launch materials for the vertical slice.

Total Capital Range

$4,800 – $15,600

Combined setup + demo production target for this phase.

How the Demo Fits the Bigger Plan

  • Phase 1 – Ship Hotdog Hustler to generate initial revenue.
  • Phase 2 – Use revenue + small additional funding to set up Kato.8 and build the Last Light demo.
  • Phase 3 – Use the demo as the core asset for full-game funding and publishing conversations.

Timeline

From Micro-Game to Demo

The demo sits in the middle of the broader funding roadmap, following the release of Hotdog Hustler and preceding full Last Light production.

0–2 Months · Hotdog Hustler Release

Finalize, launch, and begin monetizing Hotdog Hustler to create initial revenue.

2–3 Months · Studio Setup & Prototype

Apply initial funds to establish Kato.8's basic operations and begin prototyping the Last Light core loop.

3–6 Months · Last Light Demo Build

Focused development of the demo slice: core systems, art pass, and investor-ready build.

6+ Months · Fundraising With the Demo

Use the polished demo and supporting documents to secure funding for the full game.

Risks

Key Demo Risks & Mitigation

The demo plan is scoped with a clear view of risks around budget, time, and scope creep.

  • Budget risk: Hotdog Hustler revenue or small funding may land on the lower end.
    Mitigation: Plan for multiple budget scenarios and prioritize core demo systems first.
  • Time & availability: Small team with limited hours can slow demo progress.
    Mitigation: Ruthless scope control and milestone-based planning so the slice stays achievable.
  • Scope creep: Temptation to add full-game features into the demo.
    Mitigation: Lock demo scope around base building, farming, and simple waves only; full systems live on the full-game page.
