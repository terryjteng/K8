Last Light

2D Zombie Survival & Home-Fortification Game

A 2D zombie survival and home-fortification game where players gather wood, stone, and metal, upgrade a single home, build turrets, and manage a shared ammo pool between themselves and their defenses.

Studio: Kato.8
Structure: Demo → Expanded Scope → Full Game
Contact: terryjteng@gmail.com

Executive Summary

  • Core: 2D zombie survival and home-fortification focused on one upgradeable home.
  • Loop: Gather wood, stone, and metal → strengthen the home and build turrets → manage shared ammo and survive zombie threats.
  • Key Tension: The same ammo supports both the player’s gun and the home’s turrets.
  • Demo Role: Prove that this resource and ammo loop is clear, satisfying, and sustainable.
  • Full Game: Extend the same systems—more ways to gather, fortify, and decide how to use limited ammo—without changing the core identity.

Design Pillars

Home at the Center

The home is the anchor. Players always come back to this single space and invest their resources into keeping it standing.

Shared Ammo, Shared Fate

Ammo decisions affect both the player and their turrets. Spending aggressively now impacts how well the home can be defended later.

Focused Feature Set

Systems stay tightly scoped: resources, home upgrades, turrets, machete, gun, and zombies. The game grows by deepening these, not by adding unrelated features.

Core Loop: Gather • Fortify • Survive

Gather
  • Obtain wood, stone, and metal.
  • These resources are the foundation of every upgrade and turret.
Fortify
  • Spend resources to reinforce and improve the home.
  • Build and maintain turrets that support defense.
Survive
  • Use machete and gun while turrets provide supporting fire.
  • All gunfire and turret usage draw from the same ammo pool.
  • How long the home stands depends on resource and ammo choices.
Repeat
  • Gather again, patch weaknesses, and adjust how resources and ammo are used.

Systems Overview

Resources

  • Wood: General structural use and upgrades.
  • Stone: Stronger, more resilient improvements.
  • Metal: Turrets and more advanced reinforcement options.

Player Tools

  • Machete for close-range use without consuming ammo.
  • Gun for ranged use that consumes ammo.

Home & Turrets

  • One central home to protect and upgrade.
  • Turrets built around the home that use the shared ammo pool.
  • Zombie threats put pressure on how well the home is prepared.

Demo Scope

The demo focuses on proving the core loop works with a contained set of features and content.

Core Features in Demo
  • 2D zombie survival and home-fortification loop.
  • Wood, stone, and metal gathering.
  • Home upgrades using those resources.
  • Turrets that share ammo with the player’s gun.
  • Machete and gun as primary player tools.
Demo Goals
  • Show that the resource and ammo decisions feel meaningful.
  • Verify that the home-fortification focus is clear and satisfying.
  • Provide a solid reference for the production plan and investor conversations.

Full Game Vision

The full game extends the demo’s systems instead of replacing them. The same home, resources, and shared ammo remain at the center while options and variety grow.

What Stays the Same
  • 2D zombie survival and home-fortification focus.
  • Wood, stone, and metal as core resources.
  • Single central home to protect.
  • Turrets using a shared ammo pool with the player.
How It Expands
  • More ways to use wood, stone, and metal to shape the home.
  • More turret and upgrade possibilities using the same core logic.
  • Additional layers of decision-making around ammo and fortification.

Roadmap

Phase 1: Demo

  • Implement the core resource, home, turret, and ammo systems.
  • Ensure the core loop is playable and understandable.

Phase 2: Expanded Scope

  • Add more options for how the home can be improved and defended.
  • Refine how wood, stone, and metal are used.

Phase 3: Full Game

  • Finalize content, polish, and presentation.
  • Keep the focus on survival and home-fortification with shared ammo.

Exact dates, budgets, and headcount are managed in the studio’s separate financial and production planning documents and can be updated without changing this high-level structure.

Technical Overview

Foundation
  • Unity 2D project structure.
  • Separate systems for resources, home state, turrets, and ammo tracking.
Key Considerations
  • Reliable tracking of shared ammo between player and turrets.
  • Clear communication of home status and resource usage to the player.

Team & Roles

Core Roles
  • Producer / Designer – Owns scope, vision, and coordination.
  • Unity Developer – Implements systems and gameplay logic.
  • 2D Artist / UI – Visuals for home, zombies, turrets, and interface.
  • Sound / Music – Audio that supports survival and tension.
Support (As Needed)
  • Additional developers or artists depending on funding.
  • QA support for stability and feel.
  • Marketing support shared with the rest of the Kato.8 slate.

Specific rates, duration, and staffing plans are captured in separate studio-level documents.

Risk & Mitigation

Unclear Focus

Risk: Adding features that distract from home-fortification.

Mitigation: Keep every feature tied directly to resources, home, turrets, or ammo decisions.

Shared Ammo Feel

Risk: Shared ammo feels confusing or unfair.

Mitigation: Iterate on UI and feedback in the demo; keep the mechanic readable and predictable.

Production Capacity

Risk: Team size and funding slow development.

Mitigation: Use demo-first approach and align full game scope with funding scenarios.
